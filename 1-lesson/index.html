Перелік основних команд:

  • git config --global user.name blabla - задає ім'я користувача, до якого будуть прив'язуватися коміти;
  • git config --global user.email blabal@bla - задає адресу електронної пошти користувача, до якої будуть прив'язані коміти;
  • git init - створити пустий репозиторій;
  • git status - перевірка статусу файлів у репозиторії;
  • git add - додавання одного чи декількох файлів у репозиторій;
  • git commit - дозволяє при створенні коміта додати повідомлення в репозиторій з використанням параметру -m;
  • git log - перевірка змін внесених в репозиторій;
  • git checkout - дозволяє переміщуватися між гілками депозиторію;
  • git branch - створення нової гілки у репозиторії;
  • git push - відправляє зміни у віддалений репозиторій;
  • git pull - отримати зміни з віддаленого репозиторію;
  • git merge - довзоляє об'єднувати гілки в репозиторії;
  • git clone - отримання коду із віддаленого репозиторію.

Довідково про .gitignore файл

Одним із типів файлів, які розглядаються GIT є ігноровані файли - це файли, які з тих чи інших причин не повинні попадати в коміти. Ігноровані файли відслідковуються у спеціальному файлі .gitignore, який реєструється у кореневому каталозі репозиторію. У Git немає спеціальної команди для вказівки ігнорованих файлів, замість цього необхідно вручну редагувати файл .gitignore, щоб вказати у ньому файли, які повинні бути ігноровані. Файл .gitignore містить шаблони, які співставляються з іменами файлів у репозиторії для визначення необхідності ігнорувати такі файли.
